# エージェント編（koto-log）の見出し変形例

汎用テンプレの各節を koto-log 系テーマでどう具体化するかの例。

## 例1: A-1 責務分離（シリーズの看板記事）

```
## はじめに
## 題材アプリ: LINEで動く育児記録エージェント
## 課題: LLMに任せすぎると何が起きるか
   （時刻計算ミス・集計のブレ・表記ゆれ、の実例）
## 全体像: LLM担当 / アプリ担当の境界線
   （責務分担表 + シーケンス図）
## 実装
### 「さっき」はLLMに変換させない — timeparse.py
### 「粉ミルク」→「ミルク」の正規化 — subtype.py
### 計算はexecutorが確定値で行う — executor.py
## 設計判断とトレードオフ
   （全部LLMにやらせる案 / 全部ルールベースの案 との比較）
## まとめ
```

## 例3: A-6 可観測性

```
## 課題: LLMコストは「気づいたら増えている」
## 全体像: 全呼び出しの唯一の通り道で計測する
## 実装
### LLMClient.complete() に計測を集約
### contextvarで1リクエスト=1トレースに紐付け
### Sink差し替え設計（JsonLog→Langfuse）
## 設計判断: PIIをイベントに含めない / 計測失敗で本処理を止めない
```

## エージェント編で共通して使える「課題」の書き出しパターン

- 小型ローカルLLMの現実（tool_callsを外す・引数を間違える）から始める
- 「LLMは◯◯が苦手」という観察 → だからアプリ側でやる、の流れ
