```
## はじめに
## 題材アプリ: 日本語書籍のRAGパイプライン
## 課題: チャンク分割は「賢さ」を3段階から選べる
   （ヒューリスティック / semantic chunking / LLM判定）
## 全体像: 見出し境界+文字数+句点のヒューリスティック
## 実装
### Chunker ABCと差し替え可能設計
### HeuristicChunkerの分割ロジック
## 設計判断とトレードオフ
   （決定性・再現性・コスト vs 分割品質。見直しトリガも明記）
## まとめ: 「LLMは評価役として先に使う」
```

## 例2: B-4 精度改善（分割する場合は手法ごと1記事）

```
   （query→HyDE→embed→閾値→Hybrid RRF→Rerank→隣接展開 のパイプライン図）
## 実装
### <当該手法の実装>
### なぜこの位置に置くか（適用順序の理由）← この節が記事の核
## 評価: hit@k / MRR でBefore/After比較
## 設計判断: デフォルトOFFにした理由 / レイテンシとのトレードオフ
```

## RAG編で共通して使える「課題」の書き出しパターン

- 実際のクエリで観察した検索失敗例から始める
- 「開発は無料・本番は課金」の制約 → 差し替え可能設計、の流れ
- Before/Afterを数値（hit@k / MRR）で示せる記事は必ず示す
